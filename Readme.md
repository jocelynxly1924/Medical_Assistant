# 医疗助手

## 项目简介

医疗助手是一个基于大语言模型和知识库的智能医疗问答系统，旨在为用户提供专业、准确的医疗健康咨询服务。系统通过意图识别、信息补充、知识检索和智能回答生成几个步骤，为用户提供个性化的医疗健康建议。

## 核心功能

- **医疗意图识别**：自动识别用户的医疗问题类型（疾病科普、病症分析、药品信息、其他类别）
- **信息补充**：智能询问用户缺失的关键信息，确保回答的准确性
- **高风险词检测**：检测可能存在高风险情况，建议用户及时线下就医
- **医疗知识库检索**：从Huatuo医疗问答数据集中检索医疗诊断相关信息
- **药品信息查询**：从中国医药信息查询平台获取药品详细信息
- **智能回答生成**：基于检索结果生成专业、准确的医疗回答

## 技术栈

- **后端**：Flask, LangGraph, LangChain
- **前端**：HTML, CSS, JavaScript
- **模型**：Ollama (bge-m3), 通义千问 DashScope (qwen3-max)
- **向量数据库**：ChromaDB
- **工具**：BeautifulSoup (网页解析), Requests (网络请求)

## 工作流程

1. **意图识别**：识别用户的医疗问题类型
2. **信息补充**：询问用户缺失的关键信息
3. **信息精炼**：总结用户提供的信息
4. **信息检索**：从知识库和药品数据库中检索相关信息
5. **回答生成**：基于检索结果生成专业的医疗回答

## 安装与运行

### 环境要求

- Python 3.11
- Ollama (用于运行本地模型)
- DashScope API Key (用于调用大模型)

### 安装依赖

```bash
pip install flask langchain langgraph langchain-ollama langchain-openai langchain-chroma beautifulsoup4 requests
```

### 配置

1. 安装Ollama并下载模型：
   ```bash
   ollama pull bge-m3:latest
   ollama pull qwen2.5:7b-instruct-q4_K_M
   ```

2. 设置环境变量：
   ```bash
   export DASHSCOPE_API_KEY=your_dashscope_api_key
   ```

3. 导入医疗数据到ChromaDB：
   ```bash
   python huatuo_to_chromadb.py
   ```

### 运行

```bash
python app.py
```

系统将在 `http://localhost:5001` 运行。

## 使用指南

1. 打开浏览器，访问 `http://localhost:5001`
2. 在输入框中输入您的医疗问题
3. 系统会自动识别您的问题类型，并可能询问您一些关键信息
4. 系统会从医疗知识库和药品数据库中检索相关信息
5. 系统会基于检索结果生成专业、准确的医疗回答
6. 如果系统检测到高风险情况，会建议您及时线下就医

## 示例

### 疾病科普

**用户**：什么是感冒？
**系统**：感冒是一种常见的上呼吸道感染疾病，主要由病毒引起，常见症状包括鼻塞、流涕、咳嗽、喉咙痛等。感冒通常具有自限性，一般在7-10天内可自行恢复。

### 病症分析

**用户**：我咳嗽五天没好，该怎么办？
**系统**：请问您的年龄是多少？是否有其他症状，如发热、咳痰等？
**用户**：我25岁，有轻微发热，痰不多。
**系统**：根据您的症状，可能是普通感冒或急性支气管炎。建议您多休息，多喝水，可以使用一些对症治疗的药物，如解热镇痛药、镇咳药等。如果症状持续加重或超过一周，建议您及时就医。

### 药品信息

**用户**：板蓝根的用法用量是什么？
**系统**：板蓝根的用法用量如下：口服，一次1-2袋，一日3-4次。具体用法请参考药品说明书或遵医嘱。

## 数据来源

- **药品信息**：中国医药信息查询平台
